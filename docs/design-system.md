# Design System

## Purpose

This document defines the visual design principles, layout rules, spacing system, and UI consistency guidelines used across the project.

When reviewing or modifying the UI, prioritize consistency, readability, responsiveness, accessibility, and visual hierarchy.

---

# Design Philosophy

The website should feel:

* Modern
* Premium
* Spacious
* Professional
* Easy to scan
* Conversion-oriented

Prefer simplicity and consistency over visual complexity.

Avoid introducing visual elements that compete with the primary content or call-to-actions.

---

# Layout Rules

## Hero Sections

Hero sections are the main visual entry point of a page.

Guidelines:

* Maximum width: 1600px
* Full-width backgrounds are allowed
* Content should remain visually centered
* Avoid extremely long text lines
* Hero content should be immediately understandable

Examples:

* Video hero
* Full-width image hero
* Company presentation hero

---

## Content Sections

Standard content containers should use:

* Maximum width: 1200px

This width should be used for:

* Service sections
* Feature sections
* Card grids
* Company information
* Contact sections

Avoid arbitrary container widths.

---

## Reading Content

Long text content should remain highly readable.

Use:

* Maximum width: 800px

Examples:

* Blog articles
* Long descriptions
* Company story sections
* Legal pages

---

# Container Consistency

Preferred hierarchy:

Hero:

* 1600px

Content:

* 1200px

Reading:

* 800px

Avoid mixing multiple container widths unless there is a clear visual reason.

---

# Spacing System

Use the following spacing scale consistently.

xs:

* 8px

sm:

* 16px

md:

* 24px

lg:

* 48px

xl:

* 80px

xxl:

* 120px

---

# Section Spacing

Recommended vertical spacing:

Small sections:

* 48px top
* 48px bottom

Standard sections:

* 80px top
* 80px bottom

Hero transitions:

* 120px top
* 120px bottom

Avoid inconsistent spacing between sections.

---

# Typography

## Headings

Requirements:

* Clear hierarchy
* Consistent scale
* Strong visual contrast

Recommended:

H1:

* One per page

H2:

* Major section titles

H3:

* Subsections

---

## Paragraphs

Guidelines:

* Comfortable reading width
* Adequate line height
* Avoid large blocks of text

Recommended:

* Line height: 1.5–1.8
* Paragraph spacing: 16–24px

---

# Visual Hierarchy

Each section should clearly communicate:

1. What is this?
2. Why is it important?
3. What should the user do next?

Ensure the primary CTA is visually dominant.

---

# Images

Requirements:

* High quality
* Consistent aspect ratios
* Responsive sizing

Avoid:

* Pixelated images
* Mixed image styles
* Excessive decorative imagery

---

# Video Usage

Videos should enhance the experience without blocking content.

Guidelines:

* Use poster images
* Optimize file size
* Avoid autoplay audio
* Mobile experience must remain smooth

Hero videos may use wider containers than normal content sections.

---

# Responsive Design

The site must work well on:

* Mobile
* Tablet
* Desktop
* Large desktop displays

Review:

* Typography scaling
* Button sizes
* Spacing
* Layout wrapping
* Navigation behavior

---

# Accessibility

Minimum requirements:

* Semantic HTML
* Keyboard navigation
* Visible focus states
* Sufficient contrast
* Accessible forms
* Alt text for meaningful images

---

# UI Review Checklist

Before approving UI changes verify:

* Layout consistency
* Container width consistency
* Typography hierarchy
* Responsive behavior
* Accessibility
* CTA visibility
* Spacing consistency
* Visual balance

Any deviation from these rules should be justified in the implementation notes.
